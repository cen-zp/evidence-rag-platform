# 3–5 分钟演示脚本

本脚本用于录制真实本地工作台演示。它是录制提纲，不是已经完成的视频；录制后把视频放在私有演示链接或发布页，并在 README 增加链接。单次演示只证明链路可用，不是稳定性能或质量结论。

## 录制前检查

1. `docker compose ps` 中 API、Worker、PostgreSQL、Redis、Qdrant 和 Web 正常运行。
2. 浏览器已登录本地账户，页面顶部显示“API 已连接”。
3. 准备一个**非简历指标**演示知识库，至少有两份状态为 `ready` 的文档；演示题集只能展示功能，不得展示为正式效果结果。
4. 关闭终端、`.env`、密码管理器和任何含敏感信息的窗口。
5. 确认 Reranker 已启用，`RETRIEVAL_MIN_SCORE=0.02`；不要为了录制速度关闭重排序或低置信度守卫。
6. 预先准备两个问题：一个能由演示资料直接回答，一个主题明确不在资料中。正常回答会调用模型并产生少量费用；录制前需获得本次调用授权。

## 镜头与讲解（约 4 分钟）

| 时间 | 屏幕操作 | 建议讲解 |
| --- | --- | --- |
| 0:00–0:25 | 打开工作台，展示“API 已连接”和已登录账户。 | “这是一个面向内部资料的可评测证据问答平台，模型只在检索到资料后回答。” |
| 0:25–0:55 | 新建演示知识库或切换到已有演示库，展示资料列表。 | “知识库、文档和会话按账户隔离；每个文档都有异步处理状态。” |
| 0:55–1:35 | 上传一份小 Markdown/PDF/DOCX，等待或展示其 `ready` 状态。 | “上传只创建任务，ARQ Worker 负责解析、分块、BGE 向量化并写入 Qdrant。” |
| 1:35–2:20 | 提一个确实能由资料回答的问题，展示“正在检索证据并生成回答”、模型名、token、证据数，以及检索、服务端管线和浏览器端到端三类耗时。 | “检索融合 BGE、BM25、RRF，并用 CrossEncoder 重排序；模型返回结构化引用，服务端只接受本轮命中的 chunk。几类耗时分列记录，不用模型耗时冒充端到端延迟。” |
| 2:20–2:45 | 展示右侧来源片段、文件名和页码/位置；点击“有帮助”。 | “这里展示的证据来自服务端校验；反馈绑定到持久化回答，便于后续分析。” |
| 2:45–3:15 | 新建对话或打开会话记录，展示可继续追问。 | “后续追问读取服务器保存的最近会话上下文，但每轮仍重新检索，不跨知识库带入上下文。” |
| 3:15–3:40 | 提一个明确不在资料中的问题，展示 `retrieval-guard` 拒答与本次耗时。 | “当 Reranker 候选全部低于校准阈值 0.02 时，服务端在模型调用前拒答，不产生这次模型费用；非法引用也会被服务端拒绝。” |
| 3:40–4:10 | 打开评测面板，展示检索指标、人工评审、模型用量和端到端延迟汇总。 | “汇总区分正常回答与守卫拒答，并分别记录检索、服务端管线、浏览器端到端样本。两条演示样本只证明观测链路，不能代表稳定 P95。” |
| 4:10–4:30 | 展示验证文档中的正式题集与限定说明。 | “旧 72 题批次由一位评审者声明式逐题审核；24 条越界题是开发者声明式校准集。项目不会把它们夸大成独立第三方或多评审结论。” |

## 录制后的核对

- 视频中可看到：登录状态、文档处理、正常带证据回答、拒答、会话/反馈、评测入口。
- 正常回答显示：模型耗时、检索耗时、服务端问答管线耗时、浏览器端到端耗时；延迟汇总至少包含 1 条回答和 1 条守卫拒答。
- 视频中不出现：API Key、密码、Bearer token、`.env`、数据库连接串或真实私人资料。
- 在 README/简历中只能引用带题集、样本量、审核方法和限制说明的结果，不能引用演示题集或单次截图的数字。
- 如果浏览器端到端写回失败或汇总样本数没有增加，不把该次录制当作验收证据；先修复并重录相关片段。

## 录制产物

1. 导出 3–5 分钟 MP4，建议命名为 `evidence-rag-platform-demo-YYYYMMDD.mp4`。
2. 完整观看一遍，确认音画同步、文字可读，并再次检查没有密钥、密码、token 或私人资料。
3. 计算文件 SHA-256，并在 README 写入实际可访问的视频链接、录制日期和哈希；文件或链接不存在时不得提前写成“演示视频已提供”。
