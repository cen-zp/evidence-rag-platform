# 3–5 分钟本地 MVP 验收演示

本方案用于录制当前已实现的本地 MVP。默认路径**不新增任何语言模型调用**：上传、异步处理和检索评测只使用本地服务与本地 BGE；正常回答、引用和守卫拒答通过现存持久化会话或哈希绑定的固定批次回放。

视频录制完成前，本文件仍只是录制方案，不代表已经提供演示视频。一次演示只证明这条本地链路可用，不代表稳定准确率、延迟或成本分布。

## 已核对的真实页面边界

- `/`：本地注册/登录、知识库、Markdown/PDF/DOCX 上传、每 3 秒文档状态刷新、会话回看、反馈和来源证据。
- `/evaluation`：评测案例管理、本地检索评测、人工评审覆盖、历史模型用量/成本和端到端耗时汇总。
- `/review`：读取仓库内固定 72 题报告，校验 batch ID、SHA-256 和题数后展示历史回答与服务端校验引用；打开和翻题不会调用模型。
- 主工作台只逐条展示回答、反馈和来源证据；模型 token、成本和多类耗时只在评测页以历史汇总展示。
- SSE 是“检索状态 + 最终校验结果”，不是未经校验的逐 token 输出。

## 零付费操作边界

录屏中可以执行：

- 注册/登录、创建知识库、上传演示文件、等待 `等待处理 -> 处理中 -> 可检索`；
- 打开现存会话、查看现存回答/拒答/引用；
- 新增一条演示评测案例并点击“运行检索评测”；
- 打开 `/review` 查看固定批次。

录屏中不要执行：

- 点击主工作台“发送”；即使问题看似越界，只要检索仍有合格命中就可能调用 DeepSeek；
- 点击评测页“生成并评审”；该按钮会调用知识库问答流；
- 运行 `app.evaluation.answer_batch` 或任何真实回答批处理；
- 打开 `.env`、浏览器存储、开发者工具网络请求、密码管理器或包含 token/连接串的终端输出。

## 录制前准备

1. 启动现有本地栈：

   ```bash
   docker compose up --build -d --wait
   ./scripts/demo-preflight.sh
   ./scripts/demo-model-call-snapshot.sh > /tmp/evidence-rag-model-calls.before
   ```

2. 准备上传文件 [mvp-acceptance-handbook.md](../demo-assets/mvp-acceptance-handbook.md)。它是公开、合成的演示资料，不含私人信息，也不是正式评测题集。
3. 关闭 `.env`、密码管理器、聊天软件和含私人资料的窗口。给页面右上角账户邮箱加录屏遮罩；可直接把 [privacy-account-overlay.svg](../demo-assets/privacy-account-overlay.svg) 作为 OBS/剪辑软件的图片层，不要把真实邮箱当演示内容。
4. 正常回答首选打开当前本机已有会话：
   `FastAPI 的 HTTPException 和 Starlette 的 HTTPException 在异常处理器中该优先考虑哪一个？`
   已核对的回答引用 `06-handling-errors.md`。若运行数据库没有这条会话，直接使用 `/review` 的固定批次回放，不重新发送问题。
5. 守卫拒答首选打开已有会话：`今天上海的天气和空气质量怎么样？`。不要为录屏重新提交这道题。
6. 注册镜头使用 `.invalid` 域名的演示邮箱，不使用个人邮箱。密码在开始录制前私下准备，不写入脚本、不朗读；输入密码时暂停画面采集，恢复后只能看到浏览器的掩码圆点。若实际创建了新账户，前半段在该演示账户中完成建库/上传，后半段需明确切回原先已有会话的验收窗口；不要把两段说成同一账户的一条连续事务。

| 0:25–0:50 | 在已登录工作台新建 `MVP 录屏验收 YYYYMMDD` 知识库。 | “创建后资源绑定当前账户。删除知识库时会清理数据库记录、向量和上传源文件。” |
| 0:50–1:30 | 上传 `demo-assets/mvp-acceptance-handbook.md`，展示真实状态变化；若处理太快，只保留真实的上传后与 `可检索` 两个镜头，不补造 `处理中`。 | “上传先保存文件和 `pending` 记录，再由 Redis/ARQ Worker 解析、分块，用本地 BGE 建向量并写入 Qdrant；失败任务可重新入队。” |
| 1:30–2:05 | 打开 `/evaluation`，选择新库，新增问题 `文档上传后经过哪些处理状态？`，预期文件选择 `mvp-acceptance-handbook.md`，点击“运行检索评测”。 | “这一步只跑本地检索，不调用语言模型。单题结果只是功能烟雾检查，不能作为准确率或简历指标。” |
| 2:05–2:50 | 切回原先已登录的本地验收窗口，选择已有 FastAPI 知识库，打开上述 `HTTPException` 历史会话，停在回答和右侧 `06-handling-errors.md` 来源片段。不要点击“发送”。 | “这是此前真实请求持久化后的回放，不是本次录屏生成。模型引用先由服务端限制在本轮检索结果内，再随会话保存；服务端校验引用身份不等于自动证明引用充分。” |
| 2:50–3:15 | 打开已有天气会话，展示守卫拒答；同时让会话列表可见。 | “这是已有守卫结果。无合格证据时服务端会拒答；每轮仍重新检索，知识库会话最多向模型携带最近 6 条服务端上下文。” |
| 3:15–3:50 | 打开 `/evaluation`，展示 72 条现存案例、当前数据库中的在线人工评审覆盖、历史模型用量/成本与回答耗时。不要点击“生成并评审”。 | “这里把检索、人工判断、历史模型用量和端到端样本分开。在线覆盖不等于仓库中已提交的正式审核 CSV；历史数字不代表本次录屏新增调用，也不能把小样本 P95 当稳定性能。” |
| 3:50–4:20 | 打开 `/review`，展示 batch ID/报告哈希、问题、参考要点、历史模型回答和“服务端校验后的引用”。 | “这是哈希绑定的现存 72 题批次。旧质量结果来自一位评审者的声明式逐题审核，不是独立第三方或多评审结论；本页只读展示不会调用模型。” |

若创建账户、文档首次加载本地模型或检索评测耗时较长，可以剪掉等待段，但不可用动画或手改数据库伪造状态。最终视频应保持 3–5 分钟。

## 录屏完成后的最小验证

先执行：

```bash
./scripts/demo-postflight.sh \
  /absolute/path/evidence-rag-platform-demo-YYYYMMDD.mp4 \
  /tmp/evidence-rag-model-calls.before
```

然后完整观看一次，只确认以下五项：

- 时长 3–5 分钟，画面文字可读，音画同步；
- 能看到注册入口/已登录状态、建库、上传与真实处理结果、现存回答和引用、会话/拒答、评测页与 `/review`；
- 视频中没有 `.env`、密码字符、API Key、Bearer token、数据库连接串、个人邮箱或私人资料；
- 录制前后全局 `model_call_count` 与 `latest_model_call_at` 完全一致；这是“没有新增已记录成功调用”的必要条件，不覆盖失败或非法引用后的上游尝试，因此还必须确认全程没有点击“发送”/“生成并评审”或运行回答批处理；
- 讲解没有把单题烟雾检查、固定批次回放、单人审核或历史用量包装成实时结果、独立评审或稳定性能。
