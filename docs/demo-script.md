# 3–5 分钟演示脚本

本脚本用于录制真实本地工作台演示。它是录制提纲，不是已经完成的视频；录制后把视频放在私有演示链接或发布页，并在 README 增加链接。

## 录制前检查

1. `docker compose ps` 中 API、Worker、PostgreSQL、Redis、Qdrant 和 Web 正常运行。
2. 浏览器已登录本地账户，页面顶部显示“API 已连接”。
3. 准备一个**非简历指标**演示知识库，至少有两份状态为 `ready` 的文档；演示题集只能展示功能，不得展示为正式效果结果。
4. 关闭终端、`.env`、密码管理器和任何含敏感信息的窗口。

## 镜头与讲解（约 4 分钟）

| 时间 | 屏幕操作 | 建议讲解 |
| --- | --- | --- |
| 0:00–0:25 | 打开工作台，展示“API 已连接”和已登录账户。 | “这是一个面向内部资料的可评测证据问答平台，模型只在检索到资料后回答。” |
| 0:25–0:55 | 新建演示知识库或切换到已有演示库，展示资料列表。 | “知识库、文档和会话按账户隔离；每个文档都有异步处理状态。” |
| 0:55–1:35 | 上传一份小 Markdown/PDF/DOCX，等待或展示其 `ready` 状态。 | “上传只创建任务，ARQ Worker 负责解析、分块、BGE 向量化并写入 Qdrant。” |
| 1:35–2:20 | 提一个确实能由资料回答的问题，展示“正在检索证据并生成回答”、模型名、耗时、token 和证据数。 | “检索融合 BGE、BM25、RRF，并用 CrossEncoder 重排序；模型返回结构化引用，服务端只接受本轮命中的 chunk。” |
| 2:20–2:45 | 展示右侧来源片段、文件名和页码/位置；点击“有帮助”。 | “这里展示的证据来自服务端校验；反馈绑定到持久化回答，便于后续分析。” |
| 2:45–3:15 | 新建对话或打开会话记录，展示可继续追问。 | “后续追问读取服务器保存的最近会话上下文，但每轮仍重新检索，不跨知识库带入上下文。” |
| 3:15–3:40 | 提一个明确不在资料中的问题。 | “没有检索证据、或模型返回非法引用时，系统会拒答，而不是把猜测包装成资料结论。” |
| 3:40–4:10 | 打开评测面板，展示检索指标、人工评审和模型用量。 | “正式结果必须用独立 60–100 条题集、人工引用判定和当日价格快照生成；演示数据不用于简历指标。” |

## 录制后的核对

- 视频中可看到：登录状态、文档处理、正常带证据回答、拒答、会话/反馈、评测入口。
- 视频中不出现：API Key、密码、Bearer token、`.env`、数据库连接串或真实私人资料。
- 在 README/简历中只引用独立题集产生的指标，不能引用演示题集或单次截图的数字。
