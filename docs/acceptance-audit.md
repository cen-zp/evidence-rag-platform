# PRD 验收审计

最后更新：2026-07-14。本文件只记录仓库与本地运行证据；不包含 API Key、账户密码或会话 token。

## 验收结论

工程实现已经覆盖第 1 阶段的大部分功能，并通过本地 Docker、自动化测试和已部署迁移验证；但**尚不能签字为完整 PRD 验收通过**。阻塞正式签字的不是“功能演示题集”，而是独立 60–100 条题集、按题人工评审后的真实指标，以及 3–5 分钟演示视频。

## 已验证的本地运行条件

- Docker Compose API、Web、PostgreSQL、Redis 与 Qdrant 已在本机启动；最新数据库迁移为 `20260714_0007`，且 `alembic check` 无待生成操作。
- 后端全量自动化测试已通过；前端 ESLint 和 production build 已通过。
- 流式接口未登录返回 `401`，携带 `Authorization` 的浏览器 CORS 预检返回 `200`。

这些结果证明可交付工程链路，不代替正式效果指标。

## 正式验收前执行顺序

1. 选定一个非演示知识库，准备独立问题与来源标注。
2. 在本地设置正式运行当日的成本单价快照，重新启动 API。
3. 用固定知识库、固定 `top_k` 与固定 reranker 配置运行检索评测。
4. 对每题模型回答做人工答案、引用和拒答评审。
5. 导出真实报告，填写 README/简历占位指标，并录制演示视频。
