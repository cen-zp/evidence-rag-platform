# PRD 验收审计

最后更新：2026-07-14。本文件只记录仓库与本地运行证据；不包含 API Key、账户密码或会话 token。

## 验收结论

工程实现已经覆盖第 1 阶段的大部分功能，并通过本地 Docker、自动化测试和已部署迁移验证；但**尚不能签字为完整 PRD 验收通过**。独立 72 题和单人逐题人工评审已完成；仍缺采用短引用键的真实复测、可复核的批次成本快照，以及 3–5 分钟演示视频。

补充：72 条答案批次另有一份 DeepSeek 的模型辅助复核，答案 `46/49`、引用 `47/49`、拒答 `0/21`；其审核表已通过内容完整性校验，但全部标为 `model_assisted`，不是人工验收。21 条实际系统 outcome 为 `retrieval_guard_invalid_citation`，不能被简化成“检索未命中”。短 `S1/S2` 引用键已在代码中实现，但上述人工结果来自修复前的旧 UUID 引用协议，必须以新的真实批次验证改善与否。

## 正式验收前执行顺序

1. 选定一个非演示知识库，准备独立问题与来源标注。
2. 在本地设置正式运行当日的成本单价快照，重新启动 API。
3. 用固定知识库、固定 `top_k` 与固定 reranker 配置运行检索评测。
4. 以短引用键重新运行固定 72 题真实批次，再对新批次逐题做人工答案、引用和拒答评审。
5. 在报告中导出人工通过率、批次延迟与已快照成本；随后才填写 README/简历指标，并录制演示视频。
