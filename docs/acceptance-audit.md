# PRD 验收审计

最后更新：2026-07-15。本文件只记录仓库与本地运行证据；不包含 API Key、账户密码或会话 token。

## 验收结论

工程实现已经覆盖第 1 阶段的大部分功能，并通过本地 Docker、自动化测试和已部署迁移验证；但**尚不能签字为完整 PRD 验收通过**。独立 72 题、旧 UUID 批次的单人逐题人工评审、短引用键真实复测和可复核成本快照已经具备；仍缺短引用键批次的逐题人工评审、浏览器端端到端延迟、经独立题集校准的低置信度阈值，以及 3–5 分钟演示视频。

## 已验证的本地运行条件

- Docker Compose API、Web、PostgreSQL、Redis 与 Qdrant 已在本机启动；最新数据库迁移为 `20260715_0008`。
- 当前代码在不含 `.env` 的隔离临时目录完成后端全量回归：59/59 通过，Ruff 通过；前端 ESLint 和 production build 通过。
- 流式接口未登录返回 `401`，携带 `Authorization` 的浏览器 CORS 预检返回 `200`。

这些结果证明可交付工程链路，不代替正式效果指标。

## 正式验收前执行顺序

1. 通过 `/review` 对 batch ID `02dbf511…` 的固定 72 题报告逐题做人工答案、引用和拒答评审。
2. 用 `answer_review` 将导出的 CSV 与报告逐字段校验，记录报告与审核表 SHA-256。
3. 在浏览器端记录完整请求的端到端延迟，并明确与模型完成耗时、检索耗时分列。
4. 用独立题集校准低置信度拒答规则，避免直接把 RRF/CrossEncoder 分数当概率阈值。
5. 更新最终人工通过率与验收结论，录制并核对 3–5 分钟演示视频。
